Structure d'un Playbook

--- - name: Déploiement Nginx hosts: web # groupe inventaire become: true # sudo gather_facts: true vars: nginx_port: 80 app_name: monapp vars_files: - vars/secrets.yml pre_tasks: - name: Mise à jour apt ansible.builtin.apt: update_cache: true tasks: - name: Installer nginx ansible.builtin.apt: name: nginx state: present notify: Restart nginx - name: Copier la config ansible.builtin.template: src: nginx.conf.j2 dest: /etc/nginx/nginx.conf mode: '0644' handlers: - name: Restart nginx ansible.builtin.service: name: nginx state: restarted post_tasks: - name: Vérifier nginx ansible.builtin.uri: url: http://localhost:{{ nginx_port }} status_code: 200

Conditions & Boucles

# when — condition - name: Installer sur Debian apt: name: curl when: ansible_os_family == "Debian" # when multiple when: - ansible_distribution == "Ubuntu" - ansible_distribution_major_version | int >= 20 # loop - name: Créer des users user: name: "{{ item }}" state: present loop: - alice - bob - charlie # loop avec dict - name: Créer fichiers file: path: "{{ item.path }}" mode: "{{ item.mode }}" loop: - { path: /tmp/a, mode: '0755' } - { path: /tmp/b, mode: '0644' }

Register & Debug

- name: Récupérer version node command: node --version register: node_version changed_when: false ignore_errors: true failed_when: node_version.rc != 0 - name: Afficher résultat debug: msg: "Node version: {{ node_version.stdout }}" - debug: var: node_version # dump entier # Accès aux propriétés register # .stdout .stderr .rc .stdout_lines # .changed .failed .results (loop) - name: Agir si installé debug: msg: "OK" when: node_version.rc == 0

Gestion des Erreurs

# Bloc try/rescue/always - block: - name: Task risquée command: /bin/might_fail - name: Suite si succès debug: msg: "Succès !" rescue: - name: Gestion de l'erreur debug: msg: "Erreur: {{ ansible_failed_result }}" always: - name: Toujours exécuté debug: msg: "Nettoyage..." # any_errors_fatal — stopper TOUS les hosts any_errors_fatal: true # max_fail_percentage max_fail_percentage: 20

group_vars & host_vars

# Structure recommandée inventory/ ├── hosts.yml ├── group_vars/ │ ├── all.yml # toutes les machines │ ├── all/ │ │ ├── main.yml │ │ └── vault.yml # secrets chiffrés │ ├── web.yml │ └── db.yml └── host_vars/ ├── web1.example.com.yml └── db1.example.com/ ├── main.yml └── vault.yml
Les dossiers group_vars/all/ permettent de séparer vars normales et vault.

roles/ └── nginx/ ├── tasks/ │ └── main.yml # tasks principales ├── handlers/ │ └── main.yml # handlers ├── templates/ │ └── nginx.conf.j2 ├── files/ │ └── index.html ├── vars/ │ └── main.yml # vars haute priorité ├── defaults/ │ └── main.yml # vars basse priorité ├── meta/ │ └── main.yml # dépendances, info └── README.md

Ansible Vault — Secrets & Chiffrement

Commandes Vault

ansible-vault

Créer & Éditer

ansible-vault create secrets.yml
Créer nouveau fichier chiffré
ansible-vault edit secrets.yml
Éditer en place
ansible-vault view secrets.yml
Afficher déchiffré
ansible-vault encrypt vars.yml
Chiffrer fichier existant
ansible-vault decrypt secrets.yml
Déchiffrer (attention !)
ansible-vault rekey secrets.yml
Changer le mot de passe

Valeur unique

ansible-vault encrypt_string 'monSecret' --name 'db_password'
Chiffrer une seule valeur
ansible-vault encrypt_string --stdin-name api_key
Depuis stdin (invisible)
# Résultat dans vars : db_password: !vault | $ANSIBLE_VAULT;1.1;AES256 3661363532356363...

Multi-passwords (IDs)

ansible-vault create --vault-id prod@prompt prod.yml ansible-vault create --vault-id dev@prompt dev.yml
ansible-playbook site.yml \ --vault-id prod@~/.vault_prod \ --vault-id dev@~/.vault_dev

Vault — Intégration

# group_vars/all/vault.yml (chiffré) vault_db_password: SuperSecret123! vault_api_key: sk-abc1234... # group_vars/all/main.yml (non chiffré) db_password: "{{ vault_db_password }}" api_key: "{{ vault_api_key }}"

Fournir le password :

ansible-playbook site.yml --ask-vault-pass
ansible-playbook site.yml \ --vault-password-file ~/.vault_pass
export ANSIBLE_VAULT_PASSWORD_FILE=~/.vault_pass
# ansible.cfg [defaults] vault_password_file = ~/.vault_pass
⚠ Ajouter *.vault.yml et .vault_pass au .gitignore !

Vault — Script & Best Practices

#!/usr/bin/env python3 # vault_pass_script.py — récupère depuis env/KMS/etc. import os, sys print(os.environ['MY_VAULT_PASS'])
ansible-playbook site.yml \ --vault-password-file ./vault_pass_script.py
✓ DO

Préfixer vars vault avec vault_

Séparer vault.yml et main.yml

Utiliser vault-id pour multi-env

Stocker password dans un secrets manager

✗ DON'T

Ne jamais commit de fichier déchiffré

Éviter --decrypt sans raison

Ne pas hardcoder les passwords
